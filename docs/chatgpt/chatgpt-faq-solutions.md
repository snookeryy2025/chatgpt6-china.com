**症状：**
- 发送消息后无响应
- 一直显示"Typing..."

1. 服务器负载高
2. 网络连接不稳定
3. 请求过于复杂
4. 浏览器缓存问题

**解决方案：**

#### 方案1：刷新页面
```
1. 刷新浏览器（F5 或 Cmd+R）
2. 如果无效，硬刷新（Ctrl+F5）
3. 重新发送消息
```

#### 方案2：简化请求
```
如果请求很复杂：
1. 分解为多个简单问题
2. 减少单次输入的内容
3. 避免过长的上下文
```

#### 方案3：检查网络
```
1. 测试网络速度
2. 切换到更稳定的网络
3. 如果使用代理，尝试更换节点
```

#### 方案4：清除缓存
```
1. 清除浏览器缓存
2. 清除 Cookie
3. 重新登录
```

#### 方案5：使用镜像站（推荐）
```
国内镜像站通常响应更快
推荐：https://xsimplechat.com
```

### ❌ 问题8：ChatGPT 回答被截断

**症状：**
- 回答突然停止
- 显示到一半就结束
- 代码只生成了一部分

**原因：**
1. 达到单次回复长度限制（约 500-1000 tokens）
2. 生成过程中出现错误
3. 网络中断

**解决方案：**

#### 方案1：让它继续
```
在对话框输入以下任一指令：
- "继续"
- "请继续"
- "continue"
- "接着往下写"

ChatGPT 会从断点继续生成
```

#### 方案2：优化请求
```
如果需要长篇内容：
1. 分段请求
   "先写第一部分：引言"
   "再写第二部分：正文"
   
2. 明确字数限制
   "请用500字简要说明"
   
3. 要求大纲后逐部分展开
   "先给我一个大纲"
   "详细展开第一部分"
```

#### 方案3：使用 GPT-4 或更高版本
```
更高版本的模型单次回复长度更长
Plus 会员可使用 GPT-4
```

**症状：**

- 不显示上传功能

**原因：**
1. 使用的是免费版（不支持文件上传）
2. 文件格式不支持
3. 文件过大
4. 浏览器兼容性问题

```

- ChatGPT Plus 会员
- GPT-4 或更高版本

免费版（GPT-3.5）不支持文件上传
```

```
支持的文件类型：
✅ PDF
✅ TXT
✅ Word (.docx)
✅ Excel (.xlsx)
✅ 图片 (JPG, PNG)

文件大小限制：
- 一般文件：< 25MB
- 图片：< 20MB

不支持：
❌ 压缩包 (.zip, .rar)
❌ 可执行文件 (.exe)
❌ 视频文件（大部分）
```

```
如果无法上传：

```

**症状：**
```
"I cannot help with that"
"这违反了我们的使用政策"
"我不能提供此类信息"
```

**原因：**
1. 请求涉及敏感话题
2. 违反 OpenAI 使用政策
3. 可能有害的内容
4. 版权保护内容

```
❌ 非法活动
❌ 暴力和仇恨内容
❌ 儿童安全
❌ 自我伤害
❌ 欺诈和诈骗
❌ 成人内容
❌ 政治敏感话题
❌ 隐私侵犯
```

**解决方案：**

#### 方案1：重新表述问题
```
不要直接询问敏感内容
改为学术或理论讨论：

❌ "如何制作XX"
✅ "请从技术角度解释XX的原理"

❌ "帮我写一封诈骗邮件"
```

```
说明你的正当使用目的：

需要了解相关背景知识，仅用于教育目的。"
```

#### 方案3：分解问题
```
将敏感问题拆分为多个中性问题
从不同角度获取信息
```

```
不同 AI 的限制策略不同
```

**症状：**
```
"Your account has been deactivated"
"This account has been blocked"
无法登录
```

**常见原因：**
1. ❌ 违反使用政策
2. ❌ 使用不当注册方式
3. ❌ 多次触发安全机制
4. ❌ 账号共享或出售
5. ❌ 异常使用模式
6. ❌ 使用临时手机号注册

**解决方案：**

#### 步骤1：查看封禁原因
```
1. 检查注册邮箱，查找 OpenAI 的通知邮件

3. 查看是否为临时封禁或永久封禁
```

#### 步骤2：申诉流程
```
如果认为封禁有误：

1. 访问 help.openai.com
2. 选择 "Account Issue"
3. 提交申诉，说明：
   - 账号邮箱
   - 封禁情况
   - 使用场景
   - 认为不应封禁的理由
   
4. 等待回复（通常 3-7 个工作日）
```

#### 步骤3：预防措施
```
避免再次被封：
✅ 遵守使用政策
✅ 使用稳定的 IP 地址
✅ 不要分享账号
✅ 不要频繁切换设备
✅ 合理使用，避免滥用
```

### ❌ 问题17：API 调用速率限制

**症状：**
```
"Rate limit exceeded"
"429 Too Many Requests"
"You exceeded your current quota"
```

**原因：**
OpenAI 对 API 调用有频率限制：

| 账户类型 | 限制（GPT-3.5） | 限制（GPT-4） |
|---------|----------------|--------------|
| 免费试用 | 3 RPM | 不可用 |
| Pay-as-you-go | 60 RPM | 10 RPM |
| 付费用户（T1） | 3,500 RPM | 40 RPM |
| 付费用户（T5） | 10,000 RPM | 500 RPM |

RPM = Requests Per Minute（每分钟请求数）

**解决方案：**

#### 方案1：添加延迟
```python
import time
import openai

for i in range(10):
    response = openai.ChatCompletion.create(...)
    print(response)
    
    # 添加延迟避免超限
    time.sleep(1)  # 延迟1秒
```

#### 方案2：使用指数退避
```python
import time
import openai

def api_call_with_retry(max_retries=5):
    for i in range(max_retries):
        try:
            response = openai.ChatCompletion.create(...)
            return response
        except openai.error.RateLimitError:
            if i < max_retries - 1:
                wait_time = (2 ** i)  # 指数增长：1, 2, 4, 8...
                print(f"Rate limit hit, waiting {wait_time}s...")
                time.sleep(wait_time)
            else:
                raise
```

#### 方案3：批量处理
```python
# 合并多个请求为一个
prompt = "请分别回答以下问题：\n1. 问题1\n2. 问题2\n3. 问题3"
response = openai.ChatCompletion.create(
    model="gpt-3.5-turbo",
    messages=[{"role": "user", "content": prompt}]
)
```

#### 方案4：升级账户等级
```
使用更多 API，提升账户等级
OpenAI 根据消费金额自动调整限制
```
