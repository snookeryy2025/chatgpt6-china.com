# 国内如何使用 ChatGPT

::: tip 本文导读
从零开始，手把手教您使用 ChatGPT。无论您是完全的新手还是想提升使用技巧，都能在这里找到答案。
:::

## 快速开始

### 访问 ChatGPT

有两种主要方式访问 ChatGPT：

#### 方式一：国内镜像网站（推荐）

**优点：** 无需翻墙，注册简单，速度快

推荐平台：
- [lazymanchat.com](https://lazymanchat.com) - 综合首选
- [chat.huoyachat.com](https://chat.huoyachat.com) - 官网平替

**使用步骤：**
1. 访问镜像网站
2. 使用邮箱注册（或免注册直接使用）
3. 开始对话

#### 方式二：ChatGPT 官网

**优点：** 官方正版，功能最全

**使用步骤：**
1. 访问 [chat.openai.com](https://chat.openai.com)（需翻墙）
2. 注册账号（需要海外手机号）
3. 选择免费版或 Plus 版本

## 提问技巧

### 基本原则

好的问题 = 好的答案

**SMART 提问法：**
- **S**pecific（具体明确）
- **M**easurable（可衡量）
- **A**ctionable（可执行）
- **R**elevant（相关性强）
- **T**ime-bound（有时间限制）

### 提问公式

#### 公式一：角色 + 任务

```
你是一位资深的 Python 程序员，
请帮我写一个读取 CSV 文件的函数。
```

#### 公式二：背景 + 需求 + 要求

```
我正在准备一个关于人工智能的演讲（背景），
需要一个10分钟的演讲稿（需求），
要求内容通俗易懂，适合没有技术背景的听众（要求）。
```

#### 公式三：完整版

```
你是一位经验丰富的旅行规划师（角色），
我计划下个月去日本东京（背景），
请帮我制定一个5天4夜的旅行计划（任务），
要求包含必去景点、美食推荐、住宿建议（要求），
预算控制在每天500元人民币以内（限制），
我喜欢文化体验和美食，不喜欢购物（偏好）。
```

## 高级技巧

### 1. 上下文管理

ChatGPT 会记住对话历史，善用这一点：

```markdown
# 第一轮对话
请介绍一下机器学习的基本概念。

# 第二轮对话（ChatGPT 会记住之前的上下文）
那么深度学习和你刚才说的机器学习有什么区别？

# 第三轮对话
请用一个实际项目的例子来说明。
```

### 2. 分步引导

对于复杂任务，分步进行：

```markdown
# Step 1
我想创建一个个人博客，请先帮我规划网站结构。

# Step 2
好的，现在请帮我设计首页的布局。

# Step 3
请为首页写一段吸引人的介绍文案。
```

### 3. 要求特定格式

明确指定输出格式：

```markdown
请分析这个产品的优缺点，
用表格形式展示，包含以下列：
- 维度
- 优点
- 缺点
- 评分（1-5星）
```

### 4. 角色扮演

让 ChatGPT 扮演特定角色：

```markdown
请扮演一位严格的代码审查员，
审查以下代码并指出潜在问题：
[粘贴代码]
```

### 5. 示例驱动

提供示例说明你的需求：

```markdown
我需要生成类似下面这样的产品描述：

【示例】
"这款咖啡豆来自埃塞俄比亚高原，
海拔1800米，手工采摘，
带有浓郁的果香和巧克力余味，
适合追求品质的咖啡爱好者。"

请为我的有机绿茶生成类似风格的描述。
```

## 最佳实践

### ✅ 推荐做法

1. **清晰表达需求**
   - 提供足够的背景信息
   - 明确说明期望的输出格式

2. **迭代优化**
   - 如果第一次回答不满意，追问
   - 说明哪里需要改进

3. **验证结果**
   - 对于重要信息，进行交叉验证
   - 代码一定要测试

4. **保存有用的对话**
   - 记录好的提示词模板
   - 收藏优质回答

### ❌ 避免做法

1. **过于模糊的问题**
   ```
   ❌ 帮我写个东西
   ✅ 帮我写一篇800字的产品介绍文章
   ```

2. **一次性要求太多**
   ```
   ❌ 帮我设计整个网站、写所有代码、准备商业计划书
   ✅ 先帮我设计网站首页的布局
   ```

3. **没有验证就使用**
   - 特别是代码、数据、事实性信息

4. **输入敏感信息**
   - 不要输入密码、身份证号等

## 模型选择

### GPT-3.5 vs GPT-4 vs GPT-4o

| 模型 | 速度 | 质量 | 成本 | 适用场景 |
|------|------|------|------|----------|
| GPT-3.5 | 最快 | 较好 | 免费 | 简单问答、日常对话 |
| GPT-4 | 较慢 | 优秀 | 付费 | 复杂任务、专业内容 |
| GPT-4o | 快 | 最优 | 付费 | 多模态任务、高要求场景 |

**建议：**
- 日常问答 → GPT-3.5
- 重要工作 → GPT-4 或 GPT-4o
- 图片理解 → GPT-4o

## 故障排除

### 常见问题

1. **回答不准确怎么办？**
   - 提供更多上下文
   - 换个方式提问
   - 明确指出错误之处

2. **对话中断了怎么办？**
   - 重新开始对话
   - 简要说明之前的内容
   - 使用"继续"命令

3. **达到使用限制？**
   - 等待一段时间
   - 考虑升级到 Plus
   - 使用国内镜像站点
